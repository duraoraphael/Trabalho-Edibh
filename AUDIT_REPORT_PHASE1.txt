================================================================================
PROBLEMAS DE SEGURANÇA
================================================================================

[CRÍTICO] SECURITY #1: JWT_SECRET padrão = "dev-secret" em produção
- Arquivo: backend/app/config.py
- Problema: Secret key hardcoded e trivial
- Risco: Qualquer um consegue assinar tokens válidos
- Recomendação: Usar variável de ambiente obrigatória em produção
- Severidade: CRÍTICO

[CRÍTICO] SECURITY #2: get_current_user retorna usuário anônimo por padrão
- Arquivo: backend/app/auth/dependencies.py
- Problema: Se não enviar token, sistema aceita como "anon@local"
- Risco: Qualquer um acessa sistema sem autenticação
- Recomendação: Diferenciar modo desenvolvimento de produção
- Severidade: CRÍTICO

[CRÍTICO] SECURITY #3: Endpoints /register acessível sem autenticação
- Arquivo: backend/app/routes/auth.py
- Problema: Qualquer um pode criar conta
- Risco: Spam, força bruta, criação de contas falsas
- Recomendação: Implementar validação de email ou restrição de criação
- Severidade: CRÍTICO

[ALTO] SECURITY #4: CORS com allow_methods=["*"] e allow_headers=["*"]
- Arquivo: backend/app/main.py
- Problema: Muito permissivo
- Risco: Possível CSRF ou ataque cross-origin
- Recomendação: Específificar métodos e headers necessários
- Severidade: ALTO

[ALTO] SECURITY #5: Token armazenado em localStorage (XSS vulnerability)
- Arquivo: frontend/src/pages/LoginPage.tsx
- Problema: JWT no localStorage é acessível via JavaScript
- Risco: XSS pode roubar tokens
- Recomendação: Usar httpOnly cookies
- Severidade: ALTO

[MÉDIO] SECURITY #6: forgot_password e reset_password são placeholders
- Arquivo: backend/app/routes/auth.py linhas 91-107
- Problema: Funções não implementadas
- Risco: Usuários podem estar em estado de insegurança
- Recomendação: Implementar fluxo seguro de reset com token temporal
- Severidade: MÉDIO

PROBLEMAS DE PERFORMANCE
================================================================================

[ALTO] PERFORMANCE #1: DashboardPage faz polling a cada 60 segundos
- Arquivo: frontend/src/pages/DashboardPage.tsx
- Problema: setInterval sem verificar se componente está montado
- Risco: Memory leak, requisições continuam após desmontar página
- Recomendação: Usar useEffect cleanup, considerar WebSocket
- Severidade: ALTO

[ALTO] PERFORMANCE #2: local_storage.py lê e escreve arquivo JSON completo
- Arquivo: backend/app/data/local_storage.py
- Problema: read_json() e write_json() fazem I/O em cada operação
- Risco: Escalabilidade péssima com muitos registros
- Recomendação: Usar cache em memória ou database real
- Severidade: ALTO

[MÉDIO] PERFORMANCE #3: Sem paginação em HistoryPage
- Arquivo: frontend/src/pages/HistoryPage.tsx
- Problema: Carrega todos os registros de uma vez
- Risco: Interface travada com >1000 registros
- Recomendação: Implementar paginação ou lazy loading
- Severidade: MÉDIO

[MÉDIO] PERFORMANCE #4: SharePointService lista todos antes de filtrar
- Arquivo: backend/app/services/sharepoint_service.py
- Problema: list_reports() retorna tudo, filtros aplicados no frontend/rota
- Risco: Escalabilidade ruim
- Recomendação: Passar filtros para serviço, executar no backend
- Severidade: MÉDIO

[BAIXO] PERFORMANCE #5: Sem memoização em componentes React
- Arquivos: Todas as páginas
- Problema: Re-renders desnecessários
- Risco: Degradação em interfaces com muitos dados
- Recomendação: Usar React.memo() e useMemo()
- Severidade: BAIXO
